# CLAUDE.md — The Cat's Pajamas Club

Сайт джаз-бара: React 18 + Vite, чистый CSS (весь в `src/index.css`, ~2000 строк),
Telegram-бот на grammY, Supabase. Лендинг устроен как «книга»: 8 полноэкранных
страниц (`hero/about/menu/events/gallery/team/booking/contacts`) с 3D-перелистыванием
(`rotateY`), **все 8 смонтированы в DOM одновременно** — переключается только CSS.
Это ключевой факт для всех правил ниже.

## Правила визуала — нарушение = баг, даже если «выглядит нормально»

Эти правила — не пожелания. Каждое выстрадано реальным багом в этом проекте,
который уже чинили (см. HANDOFF_REDESIGN.md в корне репо).

### Анимации
- Анимировать только `transform` и `opacity`. Никаких transition на
  width/height/top/left/margin/box-shadow/filter.
- **Никакого постоянного `will-change`.** 8 страниц всегда в DOM — постоянный хинт
  заставляет компоузер держать слои под все страницы сразу; это уже вызывало
  фризы и было вычищено (комментарий в `index.css` у `.book__page`). Промоушен
  происходит лениво сам при старте transform/opacity перехода.
- **Infinite-анимации (Ken Burns в hero, глоу, шиммер) обязаны стоять на паузе на
  неактивных страницах** — паттерн `animation-play-state: paused` уже есть в CSS,
  новые бесконечные анимации подключать к нему же.
- **`backdrop-filter` рядом с перелистыванием запрещён.** У `.nav` фон специально
  сделан 96% непрозрачным с blur всего 4px — это лечение задокументированной ряби
  при page-flip. Не увеличивать blur, не делать фон прозрачнее, не добавлять новые
  backdrop-filter на элементы, видимые во время переходов.
- У каждой новой анимации — ветка `prefers-reduced-motion` (паттерн по всему CSS).

### Layout книги
- **Политика скролла двухрежимная:** desktop (>900px) — страница влезает в вьюпорт
  без скролла, проверять на 1366×700, а не только на высоком мониторе; mobile
  (≤900px) — страницы скроллятся внутри себя (`overflow-y: auto`), потому что
  телефонный вьюпорт ~660px «дышит» адресной строкой и запрет скролла означал
  молчаливую обрезку контента (уже чинили). Не возвращать `overflow: hidden`
  на мобиле.
- **Safe-area обязательна:** в `index.html` стоит `viewport-fit=cover`, отступы
  шапки, страниц и `.book-controls` считаются через `env(safe-area-inset-*)`.
  Новые фиксированные элементы у краёв экрана — только с этими же отступами.
- **Резерв под fixed-элементы не убирать:** `.book__page` держит `padding-bottom`
  90px (desktop) / `74px + safe-area` (mobile) под `.book-controls` и
  `padding-top 64px + safe-area` (mobile) под фикс-шапку — иначе контролы и
  шапка накрывают контент страниц (уже чинили, дважды).
- Не использовать `height: 100vh` + `overflow: hidden` — hero так молча резал
  контент на экранах ниже ~950px (уже чинили). Только `min-height: 100dvh/100svh`.
- Fixed-инвентарь проекта: `.nav` (левая колонка шириной `--nav-w`),
  `.book-controls` (низ), риффл-оверлей. Новые fixed-элементы — только после
  проверки, что не пересекаются с этими зонами на трёх высотах экрана.
- `position: fixed` внутри `.book__page` не работает как ожидается (страницы под
  transform) — фиксированные элементы жить должны вне книги.
- Брейкпоинт бургер-меню — 900px. Не опускать: на 768px полная навигация не
  помещается и переносится (уже чинили, порог поднимали с 680).

### Дизайн-система (тема Violet Night)
- Цвета/отступы/радиусы — только токены из `[data-theme]`: `--bg`, `--bg2`,
  `--accent`, `--gold`, `--text`, `--text-dim`, `--text-mid`, `--border`, `--glow`…
  Голый hex в правке — баг.
- Роли шрифтов (жёстко): **Baskerville** — заголовки; **Involve italic** —
  подзаголовки/теглайны; **Avenir Next** — основной текст; **Jar Binks** — ТОЛЬКО
  прямые цитаты людей (`.about__quote`, `.team__quote`). Jar Binks никогда не
  ставить на заголовки, навигацию, названия — это прямое требование заказчика.
  Кегль Jar Binks ≥ 28px: мельче он нечитаем.
- Тексты двуязычные (RU/EN) в `src/data.js` — любые переименования лейблов
  синхронно в обоих языках.

## Визуальная проверка — обязательна

После каждой значимой правки визуала: `preview_start` (конфиг `cats-pajamas-club`,
порт 5173) → скриншот → посмотреть → `preview_resize` mobile/tablet/desktop, плюс
низкая высота (~700px). Известная особенность: `preview_screenshot` здесь иногда
зависает — тогда не блокироваться, а проверять связкой `preview_inspect`
(вычисленные CSS-значения) + `preview_snapshot` (структура/текст).

Полный чеклист «готово» и методика — в скилле `web-polish`
(`references/checklist.md`). Пока чеклист не пройден, задача не «готово».

## Границы и территории

- `api/bot.js`, `api/bot-setup.js` — территория параллельной бот-сессии.
  Не трогать без явной просьбы пользователя.
- Функциональные страницы `/booking`, `/admin`, `/profile`, `/auth` — готовые
  системы; при работе над лендингом не менять (ссылки на них должны работать).
- Секции Menu/Events/Shelf/Team тянут контент из API — не заменять на хардкод.
